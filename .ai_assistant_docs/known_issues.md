# 既知の問題と対処法

## Supabase Edge Function接続エラー

### 問題: Network request failed

アプリの特定の操作（主にAIチャットやイメージ生成）時に「Network request failed」エラーが発生することがあります。
これはSupabaseの既知の問題で、Edge Functionの呼び出しの約1-2%で発生する可能性があります。

#### 症状
- ランダムに「Network request failed」エラーが表示される
- エラーはセッション間で永続せず、同じアクションを再試行すると成功することが多い
- Cloudflare CDNの間欠的な問題により、ときどき502/503/520エラーが発生する

#### 実装されている対策
1. **リトライロジック**: 自動的に最大3回まで指数バックオフで再試行します
2. **エラーハンドリングの強化**: 詳細なエラーメッセージとログ記録
3. **CORSヘッダーの最適化**: Access-Control-Max-Ageを設定して接続の安定性を向上
4. **キャッシュ制御**: キャッシュ関連のヘッダーを追加して接続の信頼性を向上

#### ユーザー向け対処法
エラーが発生した場合、以下の対応をユーザーに提案してください：

1. 操作を再試行する（アプリは自動的に3回まで再試行します）
2. デバイスのインターネット接続を確認する
3. アプリを再起動する
4. 異なるネットワーク（Wi-Fi/モバイルデータ）に切り替える
5. 数分待ってから再度試す

#### 開発者向け情報
このエラーはSupabaseのインフラストラクチャに関連する問題で、クライアント側で完全に解決することはできません。
Supabaseはこの問題を認識しており、インフラの改善に取り組んでいます。

現在の対応としては、以下のベストプラクティスを実装しています：
- 指数バックオフを用いたリトライロジック
- 詳細なエラーログ収集
- ユーザーに分かりやすいエラーメッセージの表示
- CORSとキャッシュ設定の最適化

## 画像生成クォータ問題

### 問題: プレミアムユーザーでも画像生成回数が制限される

プレミアムユーザーでも画像生成が1日5回に制限される問題がありました。
これは画像生成機能の実装で、プラン情報の反映が正しく行われていなかったためです。

## その他の既知の問題

### モバイルデバイスでのメモリ使用量

大量の画像を含むチャット履歴を長時間使用すると、一部のモバイルデバイスでメモリ使用量が増加する可能性があります。

#### 対策
- 設定画面から不要なチャット履歴を削除するようユーザーに推奨
- 将来的にはチャット履歴の自動アーカイブ機能を実装予定
