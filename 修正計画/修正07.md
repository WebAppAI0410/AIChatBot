## 7. ノートモジュール（改訂実装計画）

### 7.1 機能概要

- **ノート管理**: フォルダ階層構造でのノート整理、タグ付け。 **（DB/ストアレベルでは実装済み、UIは一部実装）**
- **エディタ**: **TipTap (ProseMirrorベース)** をWebView経由で使用するWYSIWYGエディタ。Markdownサポート、拡張性、カスタマイズ性を重視。Undo/Redo、ハイライト機能などを実装予定。
    - **AI校正モード**:
        - AIアイコンをMarkdownの段落にドラッグアンドドロップすることで自動校正開始（プロンプト入力も可能）。
        - ホバー時に校正対象の段落をハイライト表示。
        - ダブルタップで選択モードに入り、段落やノート全体を複数・全選択し、ドラッグアンドドロップで校正可能。
- **AIアシスト**: ノート全体または部分的な編集支援、質問応答。上記AI校正モードを含む。 **（計画段階、UI未実装）**
- **タグ管理**: タグの追加・編集・削除、複数ノートへの一括適用。 **（DB/ストアレベルでは実装済み、UIコンポーネントは未確認）**
- **ストレージ**: SQLiteベースのローカルストレージ（オフライン対応、コンテンツ圧縮機能あり）。 **（実装済み）**

### 7.3 データベース設計（SQLite）

*（`services/sqlite.ts` の実装に基づき、`is_compressed` カラムが `notes` テーブルに存在することを明記）*
```sql
-- フォルダテーブル
CREATE TABLE folders (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  parent_id TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (parent_id) REFERENCES folders (id) ON DELETE CASCADE
);

-- ノートテーブル
CREATE TABLE notes (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  folder_id TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  is_compressed INTEGER DEFAULT 0, -- コンテンツ圧縮フラグ
  FOREIGN KEY (folder_id) REFERENCES folders (id) ON DELETE CASCADE
);

-- タグテーブル
CREATE TABLE tags (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL UNIQUE
);

-- ノートとタグの関連テーブル
CREATE TABLE note_tags (
  note_id TEXT NOT NULL,
  tag_id TEXT NOT NULL,
  PRIMARY KEY (note_id, tag_id),
  FOREIGN KEY (note_id) REFERENCES notes (id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tags (id) ON DELETE CASCADE
);

-- 画像添付テーブル（大きなコンテンツ分離）
CREATE TABLE note_attachments (
          id TEXT PRIMARY KEY,
  note_id TEXT NOT NULL,
  content_type TEXT NOT NULL,
  data BLOB NOT NULL,
          created_at TEXT NOT NULL,
  FOREIGN KEY (note_id) REFERENCES notes (id) ON DELETE CASCADE
);

-- データベースバージョン管理テーブル
CREATE TABLE db_version (
  version INTEGER PRIMARY KEY,
  applied_at TEXT NOT NULL
);
```
