## 備考: Windows環境からのビルド・開発方針

### ネイティブモジュールの取り扱い

**開発環境(Expo Go)での制約:**
- `llama.rn`などのネイティブモジュールはExpo Goでは動作しない
- 開発中はモック実装を使用

**実装順序:**
1. まずUI部分(ダウンロード管理、状態表示)を先行実装
2. ネイティブ機能はダミー実装で置き換え
3. 最終テストはEASビルドで実施

**EASビルドプラン:**
1. 開発完了後、`expo-dev-client`を追加
2. `eas build --profile development --platform android`でテスト
3. 最終ビルドは`eas build --platform android`で実施

**モック・本番切り替え手法:**
```typescript
// 開発/本番で分岐する実装パターン
let isNativeAvailable = false;
try {
  require('llama.rn');
  isNativeAvailable = true;
} catch(e) {}

// 使用時に分岐
const runInference = isNativeAvailable ? 
  realImplementation : 
  mockImplementation;
```
