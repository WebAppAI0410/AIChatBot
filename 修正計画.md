# AI チャットボットアプリ 修正計画

## 概要

本計画は、AI チャットボットアプリの機能強化と UI/UX 改善を目的とした包括的な修正内容をまとめたものです。主な変更点は以下の通りです。

1. **価格設定とクォータ制限の更新**
2. **APIキー管理のバックエンドサービス実装**
3. **UI一貫性のためのリファクタリング**
4. **ローカルモデル管理機能**
5. **画像生成機能の強化**
6. **チャットのコピー/共有機能**
7. **AIアシスト付きノート機能**

### 1.3 クォータ管理

- **月次トークンバケツ**: プランごとの月間トークン使用量制限
- **日次画像バケツ**: `img:tier0:{uid}` & `img:tier2:{uid}`（毎日0時にリセット）
- **クォータ警告**: 残り20%で警告バナー表示
- **フォールバックロジック**: クォータ超過時に下位Tierモデルへ自動切替

## 2. APIキー管理バックエンド

### 2.1 Per-User APIキープロビジョニング

| Provider | プロビジョニング方法 | レート/コスト制限 | ローテーションAPI | 備考 |
|----------|----------------------|------------------|------------------|------|
| **OpenRouter** | `POST /api/v1/keys` | `limit` (USD) or `req_per_min` | `DELETE /api/v1/keys/{id}` | 組織クレジット継承 |
| **Cloudflare Workers AI** | **Scoped API Tokens** | "Workers KV/AI" + R2/AI権限 | `PUT /user/tokens/{id}` | 90日ごとのローテーション推奨 |
| **OpenAI** (画像) | Child key via Dashboard | 組織レベルの使用上限のみ | 手動またはSeleniumスクリプト | 手動プールからのローテーション |

### 2.2 データモデル

```sql
CREATE TABLE api_keys (
  user_id UUID PRIMARY KEY,
  or_key TEXT,
  or_key_id TEXT,
  or_expires TIMESTAMP,
  cf_token TEXT,
  cf_token_id TEXT,
  cf_expires TIMESTAMP,
  oa_key TEXT,
  oa_created TIMESTAMP
);
```

### 2.3 キーローテーション

- **OpenRouter & Cloudflare**: 自動ローテーション（有効期限の5日前、30日ごと）
- **OpenAI**: 60-90日ごと（手動プールから交換）

### 2.4 エラーハンドリング

| シナリオ | アクション |
|----------|-----------|
| キー作成失敗 | 指数バックオフ再試行、共有組織キーへのフォールバック |
| Provider 429 | Tierダウングレードまたはリクエストキュー |
| ローテーション失敗 | アラート通知、古いキーの有効期限延長 |

## 3. UI/UXリファクタリング

### 3.1 UI一貫性

- **統一トップパディング**: すべての画面で `theme.safeTop` を使用
- **集中スタイル**: 共通サイズ/カラーを `ui/theme.ts` に抽出
- **コンポーネントクリーンアップ**: インラインスタイル → Tailwind風ユーティリティクラスまたは `styled()` バリアント

### 3.2 使用量表示UI

- **Settings › Usage ペイン**:
  - モデルTierごとのトークンバー（`<ProgressRing>`）
  - 画像カウント（DALL·E / SDXL-25 / Lightning）
- **ランタイム警告**:
  - 80%クォータ → 黄色バナー「残りわずか」
  - 100% → 自動フォールバックまたは429 + アップグレードCTA

## 4. ローカルモデル管理（Qwen 3 4B GGUF）

### 4.1 実装詳細

- **ダウンロードワーカー**: RN FS → `FileSystem.documentDirectory/models/qwen3-4b.gguf`、進捗表示
- **インストールボタン**: チェックサムMD5が `local_models` テーブルと一致するまで無効
- **エンジン**: llama.cpp via react-native-llm, `n_threads=4`
- **エラー状態**: ストレージ不足/ネットワークエラー → トースト + 再試行オプション

### 4.2 ステータスバッジ

- **未DL**: ⚪️ 白色
- **DL中**: 🔄 青色（進捗%表示）
- **検証中**: 🔍 黄色
- **使用可**: 🟢 緑色
- **エラー**: 🔴 赤色（タップでエラー詳細）

## 5. 画像生成機能強化

### 5.1 チャットトグル

- 左側アイコンで**画像モード**切替
- アクティブ時、ユーザープロンプトは画像パイプラインを呼び出し、AI応答に `<ImageBubble>` を含む

### 5.2 画像タブ

- **ヘッダー**: 過去画像のMasonryギャラリー（無限スクロール）
- **フッター**: 解像度ピッカー、高解像度トグル、マスクアップロード（近日公開）を備えたチャットバー
- **ボトムナビ**: 順序: *New* · **Image** · *Note* · *Settings*

### 5.3 プラン別画像制限

| プラン | DALL·E 3 | SDXL-25step | Lightning 2-step |
|--------|---------|-------------|------------------|
| Free | 0 | 5 / day | ∞ |
| Lite | 1 / day | 15 / day | ∞ |
| Premium | 3 / day | 50 / day | ∞ |

制限超過後: フォールバックチェーン → SDXL-25 → Lightning → 429
